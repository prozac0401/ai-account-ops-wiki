## 개요

AI 교육에서 사고는 거창한 해킹만 뜻하지 않는다.

수강생이 실수로 고객정보를 붙여 넣거나, API 키를 GitHub에 올리거나, 회사 코드를 Cursor에 넣거나, 공유 링크를 외부에 보내는 것도 사고가 될 수 있으니, 유의하도록 하자.

## 사고로 볼 수 있는 상황

- 개인정보가 프롬프트에 입력됨
- 고객정보가 포함된 파일이 업로드됨
- 회사 기밀 문서가 첨부됨
- API 키, 토큰, 비밀번호가 입력됨
- Cursor에 실제 운영 레포가 열림
- ChatGPT 또는 Claude 공유 링크가 외부에 노출됨
- OpenAI API 키가 공개 저장소에 올라감
- 결제 사용량이 비정상적으로 증가함
- 수강생이 계정을 타인과 공유함

## 1차 대응 원칙

1. 더 이상 사용하지 못하게 막는다.
2. 무엇이 들어갔는지 확인한다.
3. 누가, 언제, 어디에 입력했는지 기록한다.
4. 키나 비밀번호라면 즉시 폐기한다.
5. 보안/개인정보 담당자에게 보고한다.
6. 서비스 제공자 지원팀에 삭제 또는 조사 요청이 필요한지 판단한다.

## 서비스별 즉시 조치

| 서비스 | 즉시 조치 |
|---|---|
| ChatGPT Business | 멤버 일시 제거, 문제 대화 삭제 안내, 공유 링크 확인 |
| OpenAI API | API 키 revoke, API 작업 단위 예산 중지, 사용량 확인 |
| Cursor | 팀 멤버 제거, 레포 접근 중지, Privacy Mode 확인, 민감 파일 확인 |
| Google Workspace | 계정 정지, 비밀번호 재설정, Drive 공유 링크 차단 |
| Colab | 노트북 공유 중지, Drive 파일 권한 확인, 라이선스 회수 |
| Claude | 멤버 제거, 대화 삭제 안내, Owner export 영향 확인 |

## API 키 노출 대응

API 키가 노출되면 삭제보다 먼저 폐기한다.

```text
1. 노출된 키 즉시 revoke/delete
2. 같은 권한의 새 키 발급 여부 검토
3. 사용량 확인
4. 비정상 사용 비용 확인
5. GitHub 등 공개 위치에서 삭제
6. 커밋 기록에 남았으면 history 정리 검토
7. 사고 기록 작성
```

키는 한 번 노출되면 “다시 숨겼다”고 안전해지지 않는다. 반드시 폐기한다.

## 개인정보 입력 대응

```text
1. 수강생에게 사용 중지 안내
2. 입력된 개인정보 범위 확인
3. 대화/파일 삭제 가능 여부 확인
4. 서비스별 retention 정책 확인
5. 개인정보 담당자에게 보고
6. 필요 시 정보주체 통지·신고 검토
7. 재발 방지 안내 재배포
```


## 수강생에게 알릴 간단 문구

```text
실습 중 개인정보, 고객정보, 회사 기밀, API 키를 잘못 입력했거나 입력한 것 같다면 즉시 손을 들어 교육 담당자에게 알려 주세요.
숨기면 문제가 커질 수 있습니다. 빨리 알려 주는 것이 가장 안전합니다.
```

## 자주 헷갈리는 점

### 프롬프트에 넣었다가 바로 지우면 끝인가?

아니다. 화면에서 지운 것과 서비스 백엔드에서 삭제되는 것은 다를 수 있다. 서비스별 retention 정책을 확인해야 한다.

### API 키를 GitHub에서 삭제하면 끝인가?

아니다. 커밋 기록, 포크, 캐시, 봇 수집 가능성이 있다. 키는 무조건 폐기한다.

### 수강생 실수면 조직 책임이 없나?

그렇게 보기 어렵다. 조직이 교육 환경을 제공하고 계정을 부여했다면 사전 안내와 대응 체계를 갖추는 것이 안전하다.
