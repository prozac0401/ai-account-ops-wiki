# 교육생 주의사항

## 개요

이 문서는 교육생이 AI 서비스를 사용할 때 꼭 지켜야 할 내용을 정리한 문서다.

어렵게 생각할 필요는 없다. 핵심은 **실습용 데이터만 쓰고, 민감한 내용은 넣지 않는 것**이다.

## 꼭 지켜야 할 것

### 본인 계정만 사용

다른 사람 계정으로 로그인하지 않는다. 본인에게 안내된 이메일로만 로그인한다.

### 계정 공유 금지

친구나 동료에게 계정을 빌려주지 않는다. 계정을 빌려주면 사용 기록과 책임이 섞인다.

### 실습용 데이터만 사용

교육에서 제공한 샘플 데이터만 사용한다. 실제 회사 업무 자료나 고객 자료를 가져오지 않는다.

## 입력하면 안 되는 것

다음 내용은 ChatGPT, Cursor, Claude, Colab, API 실습에 입력하지 않는다.

- 주민등록번호
- 휴대전화 번호
- 주소
- 고객명과 고객 연락처
- 사내 문서 원문
- 계약서 원문
- 견적서 원문
- 회사 비밀 정보
- API 키
- 비밀번호
- 인증 토큰
- 실제 운영 서버 정보
- 비공개 소스코드 전체

## Cursor 사용 시 추가 주의

Cursor는 코드를 읽고 제안하는 도구다. 그래서 아래를 특히 조심한다.

- 실제 업무 레포를 열지 않는다.
- 실습용 레포만 연다.
- `.env` 파일을 만들었다면 절대 공유하지 않는다.
- API 키를 코드에 직접 쓰지 않는다.
- 회사 내부 코드, 고객 데이터가 들어간 코드, 운영 로그를 넣지 않는다.

## OpenAI API 실습 시 추가 주의

API 키는 비밀번호처럼 다룬다.

- 키를 화면에 띄운 채로 공유하지 않는다.
- 키를 GitHub에 올리지 않는다.
- 키를 메신저에 보내지 않는다.
- 실습이 끝나면 안내에 따라 키를 폐기하거나 반납한다.

## 계정이 안 될 때

다음 순서로 확인한다.

1. 안내받은 이메일로 로그인했는가.
2. 초대 메일을 수락했는가.
3. 개인 워크스페이스가 아니라 교육 워크스페이스로 들어갔는가.
4. 브라우저에 다른 Google 계정이 로그인되어 있지 않은가.
5. 교육 담당자에게 이메일 주소를 정확히 알려 주었는가.

## 사고 같다고 느껴질 때

실수로 민감한 내용을 넣었거나, API 키를 노출했거나, 실제 업무 자료를 올린 것 같다면 바로 알린다.

숨기면 문제가 커진다. 빨리 알려 주는 것이 가장 안전하다.

```text
“방금 실습 중에 실제 고객정보가 들어간 파일을 올린 것 같습니다.”
“API 키를 코드에 붙여 넣은 뒤 화면 공유를 했습니다.”
“회사 레포를 Cursor에서 열었습니다.”
```

이렇게 말하면 된다.

## 교육 종료 후

교육이 끝나면 AI 서비스 접근 권한은 회수될 수 있다.

개인 계정 자체를 삭제하거나 대화를 삭제하고 싶다면 서비스별 절차를 따라야 한다. 교육 담당자에게 문의하면 안내받을 수 있다.

## 한 줄 요약

**샘플 데이터만 넣고, 진짜 정보는 넣지 않는다.**
