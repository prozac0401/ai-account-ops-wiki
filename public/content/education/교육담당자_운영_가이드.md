## 개요

교육 담당자는 AI 서비스의 모든 관리자 기능을 알 필요는 없다.

하지만 수강생이 언제 어떤 계정으로 들어오고, 무엇에 동의했고, 교육이 끝나면 누가 회수하는지는 알고 있어야 한다.

## 교육 전 할 일

### 1. 수강생 명단 수집

Google Form을 쓴다면 다음 항목을 둔다.

- 이름
- 이메일 주소
- 소속 또는 과정명
- 사용할 서비스
- 개인정보 고지 확인
- 동의 여부
- 안내문 확인 여부

이메일은 오타가 많다. 특히 `gmail.com`, `gamil.com`, 공백, 한글 입력을 확인한다.

### 2. 동의와 고지

동의서와 고지서는 계정 초대 전에 받는다.

간단히 말하면 다음 내용을 알려야 한다.

- 어떤 AI 서비스를 쓰는지
- 이메일 주소를 왜 받는지
- 프롬프트와 코드가 외부 서비스로 전송될 수 있는지
- 민감정보를 넣으면 안 되는지
- 동의하지 않으면 AI 기능을 못 쓸 수 있는지

### 3. 계정 관리자에게 전달

계정 관리자에게 전달할 때는 CSV를 정리한다.

```csv
name,email,service,course,consent,status
홍길동,hong@example.com,ChatGPT;Cursor,AI기초,yes,ready
```

### 4. 사전 안내문 발송

교육 전날 또는 당일 오전에 안내문을 보낸다.

- 로그인할 이메일
- 초대 메일 확인 방법
- 워크스페이스 선택 방법
- 금지 데이터
- 현장 문의 방법

## 교육 중 할 일

### 로그인 문제 대응

가장 흔한 문제다.

- 초대 메일을 못 받음
- 다른 이메일로 로그인함
- 개인 워크스페이스로 들어감
- 결제된 기능이 안 보임
- Google 본인 확인 요청이 뜸
- Cursor가 개인 계정으로 열림
- Cursor Desktop에 이전 계정이 남아 있음
- Colab Pro가 적용되지 않음

교육 담당자는 기술 해결을 직접 다 하지 않아도 된다. 다만 어떤 계정이 문제인지 정확히 기록해서 계정 관리자에게 전달해야 한다.

### Google 로그인 챌린지 대응

강의장 PC에서 여러 계정을 한꺼번에 로그인하면 Google의 본인 확인 요청이 뜰 수 있다. 현장에서는 공식 정책 설명과 경험칙을 구분해서 안내한다.

- 본인 확인 요청은 계정 도용이 아니라 새 PC·새 위치·반복 로그인에 따른 보안 테스트일 수 있다.
- 복구 이메일 인증을 우선 사용한다.
- 한 사람이 여러 계정을 계속 인증할 수 있다고 가정하지 않는다.
- 전화번호 하나로 여러 계정을 처리하는 방식은 실패할 수 있다.
- 로그인 후에는 Google 계정의 **보안 및 로그인** → **최근 보안 관련 활동** → **보안 관련 활동 검토**에서 방금 로그인이 맞는지 확인한다.
- **내 기기** → **기기 관리**에서 강의장 PC가 정상적으로 보이는지 확인한다.

문제 기록표에는 최소한 다음을 남긴다.

```csv
course,room,pc_no,email,challenge,method,result,checked_security_at,note
AI기초,A강의장,PC-01,user01@example.com,yes,recovery_email,success,2026-04-25 09:30,
```

### Cursor 로그인 챌린지 대응

Cursor Desktop은 같은 Windows 사용자 안에 이전 로그인 상태가 남을 수 있다. 로그인 문제를 보면 먼저 “웹 계정”과 “Desktop에 표시되는 계정”이 같은지 확인한다.

- Cursor Desktop 우측 상단 계정 또는 Settings에서 이메일을 확인한다.
- Desktop에 계정이 없으면 웹 로그인을 완료한 뒤 Desktop으로 연결한다.
- 여러 PC를 동시에 로그인하지 않고 호스트별로 1~2분 간격을 둔다.
- 여러 곳에서 동시에 로그인해 인증 요청이 뜨면 작업을 멈추고 순차 로그인으로 바꾼다.
- 휴대폰 번호 인증은 번호 1개당 여러 계정을 계속 처리할 수 있다고 전제하지 않는다.
- 계정, PC 번호, 로그인 시각, 휴대폰 인증 요청 여부를 기록한다.

### 금지 데이터 재안내

실습 직전에 다시 말한다.

```text
지금부터 사용하는 AI 서비스에는 실습용 샘플 데이터만 입력해 주세요.
개인정보, 고객정보, 회사 내부 문서, API 키, 비밀번호, 실제 소스코드는 입력하지 않습니다.
```

## 교육 후 할 일

### 회수 대상 확인

교육 종료 후 계정 관리자에게 최종 명단을 전달한다.

- 실제 참여자
- 미참여자
- 중도 이탈자
- 예외적으로 연장 사용해야 하는 사람
- 즉시 회수할 사람

### 회수 완료 확인

계정 관리자에게 회수 완료 기록을 받는다.

- ChatGPT 멤버 제거 여부
- Cursor 멤버 제거 여부
- Colab 라이선스 회수 여부
- OpenAI API 키 삭제 여부
- Google Workspace 라이선스 정리 여부

### 문의 대응

수강생이 “내 대화가 삭제되나요?”라고 물을 수 있다.

답변은 이렇게 한다.

> “교육용 서비스 접근 권한은 회수됩니다. 다만 각 서비스의 대화, 파일, 계정 삭제는 서비스별 정책에 따라 처리됩니다. 개인 계정 자체 삭제나 대화 삭제를 원하시면 별도 절차를 안내드리겠습니다.”

## 교육 담당자가 하지 말아야 할 일

- 관리자 계정을 수강생과 공유하지 않는다.
- 결제 카드를 임의로 등록하지 않는다.
- 법무 검토 전 동의서 문구를 확정했다고 말하지 않는다.
- “데이터는 무조건 삭제됩니다”라고 단정하지 않는다.
- “Privacy Mode니까 아무 데이터나 넣어도 됩니다”라고 말하지 않는다.

## 자주 헷갈리는 점

### 교육 담당자가 개인정보 고지의 법적 책임자인가?

보통 최종 책임은 조직의 개인정보 처리 체계 안에서 정해진다. 교육 담당자는 고지와 동의 절차를 실행하는 역할에 가깝다. 하지만 실행 기록은 매우 중요하다.

### 수강생이 동의하지 않으면 어떻게 하나?

AI 서비스 계정은 발급하지 않는 것이 원칙이다. 대체 실습이 있는지는 과정별로 정한다.

### 현장에서 계정이 안 되면 새 계정을 바로 만들어도 되나?

동의와 고지가 끝난 계정인지 먼저 확인한다. 급하다고 미동의 계정을 초대하면 나중에 더 곤란해진다.
