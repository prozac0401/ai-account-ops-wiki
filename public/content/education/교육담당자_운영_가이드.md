[목차]

## 개요

교육 담당자는 AI 서비스의 모든 관리자 기능을 알 필요는 없다.

하지만 수강생이 언제 어떤 계정으로 들어오고, 무엇에 동의했고, 교육이 끝나면 누가 회수하는지는 알고 있어야 한다.

## 교육 전 할 일

### 1. 수강생 명단 수집

Google Form을 쓴다면 다음 항목을 둔다.

- 이름
- 이메일 주소
- 소속 또는 과정명
- 사용할 서비스
- 개인정보 고지 확인
- 동의 여부
- 안내문 확인 여부

이메일은 오타가 많다. 특히 `gmail.com`, `gamil.com`, 공백, 한글 입력을 확인한다.

### 2. 동의와 고지

동의서와 고지서는 계정 초대 전에 받는다.

간단히 말하면 다음 내용을 알려야 한다.

- 어떤 AI 서비스를 쓰는지
- 이메일 주소를 왜 받는지
- 프롬프트와 코드가 외부 서비스로 전송될 수 있는지
- 민감정보를 넣으면 안 되는지
- 동의하지 않으면 AI 기능을 못 쓸 수 있는지

### 3. 계정 관리자에게 전달

계정 관리자에게 전달할 때는 CSV를 정리한다.

```csv
name,email,service,course,consent,status
홍길동,hong@example.com,ChatGPT;Cursor,AI기초,yes,ready
```

### 4. 사전 안내문 발송

교육 전날 또는 당일 오전에 안내문을 보낸다.

- 로그인할 이메일
- 초대 메일 확인 방법
- 워크스페이스 선택 방법
- 금지 데이터
- 현장 문의 방법

### 5. 로그인 챌린지 사전 해결

로그인 챌린지는 교육 중에 처리할 일이 아니라, 교육 이전에 계정 담당자와 과정 담당자가 미리 해결해야 할 일이다.

강의장 PC에서 계정 로그인을 관리자가 일괄 준비하는 경우가 많다. 이때 Google 본인 확인 요청이나 Cursor 인증 요청이 뜰 수 있으므로, 수업 시작 전에 좌석별 접속 테스트를 끝내고 기록을 남긴다.

Google 계정은 다음을 교육 전에 확인한다.

- 수강생 계정마다 복구 이메일이 준비되어 있는가.
- 강의장 PC에서 최초 로그인이 되는가.
- 본인 확인 요청이 뜨면 복구 이메일 인증으로 통과되는가.
- 로그인 후 Google 계정의 **보안 및 로그인** → **최근 보안 관련 활동** → **보안 관련 활동 검토**에서 방금 로그인이 맞다고 확인했는가.
- **내 기기** → **기기 관리**에서 강의장 PC가 정상적으로 보이는가.

Cursor 계정은 다음을 교육 전에 확인한다.

- Cursor Desktop에 이전 과정 또는 다른 수강생 계정이 남아 있지 않은가.
- 웹 로그인 후 Desktop으로 연결되는가.
- Desktop 우측 상단 계정 또는 Settings에 올바른 이메일과 Team이 표시되는가.
- 여러 PC를 동시에 처리하지 않고, 호스트별로 1~2분 간격을 두고 로그인했는가.
- 인증 요청이 발생한 계정, PC 번호, 인증 수단, 처리 결과를 기록했는가.

사전 점검 기록표에는 최소한 다음을 남긴다.

```csv
course,room,pc_no,email,service,challenge,method,result,checked_security_at,note
AI기초,A강의장,PC-01,user01@example.com,Google,yes,recovery_email,success,2026-04-25 09:30,
AI기초,A강의장,PC-01,user01@example.com,Cursor,no,web_to_desktop,success,2026-04-25 09:35,
```

사전 해결 완료 기준은 “로그인 화면을 한 번 봤다”가 아니다. 강의장 PC에서 실제로 필요한 서비스에 들어가고, 올바른 워크스페이스 또는 Team이 보이며, Google 보안 활동 확인까지 끝난 상태를 완료로 본다.

## 교육 중 할 일

### 로그인 문제 대응

교육 중 로그인 문제는 새로 해결할 일이 아니라, 사전 점검에서 빠진 항목을 응급 처리하는 일에 가깝다.

- 초대 메일을 못 받음
- 다른 이메일로 로그인함
- 개인 워크스페이스로 들어감
- 결제된 기능이 안 보임
- 사전 해결되지 않은 Google 본인 확인 요청이 뜸
- Cursor가 개인 계정으로 열림
- Cursor Desktop에 이전 계정이 남아 있음
- Colab Pro가 적용되지 않음

교육 담당자는 기술 해결을 직접 다 하지 않아도 된다. 다만 어떤 계정이 문제인지 정확히 기록해서 계정 관리자에게 전달해야 한다.

### 교육 중 응급 대처

수업 중 로그인 챌린지가 발생하면 전체 좌석에서 동시에 재로그인을 시도하지 않는다. 문제 계정과 PC만 분리해 처리한다.

1. PC 번호, 수강생 이메일, 서비스명, 오류 화면을 기록한다.
2. 계정 담당자에게 즉시 전달한다.
3. Google은 준비된 복구 이메일 인증을 우선 시도한다.
4. Cursor는 Desktop에 표시된 계정 이메일과 웹 로그인 계정이 같은지 먼저 확인한다.
5. 인증 요청이 반복되면 해당 PC는 잠시 중단하고, 동의와 고지가 끝난 예비 계정 또는 예비 PC가 있는 경우에만 전환한다.
6. 관리자 계정이나 다른 수강생 계정을 빌려 주지 않는다.
7. 수업 후 응급 처리 내역을 사전 점검표에 보완해 다음 회차 전에 같은 문제가 반복되지 않게 한다.

### 금지 데이터 재안내

실습 직전에 다시 말한다.

```text
지금부터 사용하는 AI 서비스에는 실습용 샘플 데이터만 입력해 주세요.
개인정보, 고객정보, 회사 내부 문서, API 키, 비밀번호, 실제 소스코드는 입력하지 않습니다.
```

## 교육 후 할 일

### 회수 대상 확인

교육 종료 후 계정 관리자에게 최종 명단을 전달한다.

- 실제 참여자
- 미참여자
- 중도 이탈자
- 예외적으로 연장 사용해야 하는 사람
- 즉시 회수할 사람

### 회수 완료 확인

계정 관리자에게 회수 완료 기록을 받는다.

- ChatGPT 멤버 제거 여부
- Cursor 멤버 제거 여부
- Colab 라이선스 회수 여부
- OpenAI API 키 삭제 여부
- Google Workspace 라이선스 정리 여부

### 문의 대응

수강생이 “내 대화가 삭제되나요?”라고 물을 수 있다.

답변은 이렇게 한다.

> “교육용 서비스 접근 권한은 회수됩니다. 다만 각 서비스의 대화, 파일, 계정 삭제는 서비스별 정책에 따라 처리됩니다. 개인 계정 자체 삭제나 대화 삭제를 원하시면 별도 절차를 안내드리겠습니다.”

## 교육 담당자가 하지 말아야 할 일

- 관리자 계정을 수강생과 공유하지 않는다.
- 결제 카드를 임의로 등록하지 않는다.
- 법무 검토 전 동의서 문구를 확정했다고 말하지 않는다.
- “데이터는 무조건 삭제됩니다”라고 단정하지 않는다.
- “Privacy Mode니까 아무 데이터나 넣어도 됩니다”라고 말하지 않는다.

## 자주 헷갈리는 점

### 교육 담당자가 개인정보 고지의 법적 책임자인가?

보통 최종 책임은 조직의 개인정보 처리 체계 안에서 정해진다. 교육 담당자는 고지와 동의 절차를 실행하는 역할에 가깝다. 하지만 실행 기록은 매우 중요하다.

### 수강생이 동의하지 않으면 어떻게 하나?

AI 서비스 계정은 발급하지 않는 것이 원칙이다. 대체 실습이 있는지는 과정별로 정한다.

### 현장에서 계정이 안 되면 새 계정을 바로 만들어도 되나?

동의와 고지가 끝난 계정인지 먼저 확인한다. 급하다고 미동의 계정을 초대하면 나중에 더 곤란해진다.
