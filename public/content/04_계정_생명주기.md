## 개요

AI 계정은 “만드는 순간”보다 “끝나는 순간”이 더 중요하다.

이 문서는 교육생 한 명이 신청서를 내고, 계정을 받고, 교육 중 사용하고, 교육 종료 후 권한이 회수되는 전체 흐름을 설명한다.

## 전체 흐름

```text
신청/수집
  ↓
검증
  ↓
고지/동의 확인
  ↓
초대/라이선스 할당
  ↓
접속 테스트
  ↓
교육 중 사용
  ↓
권한 회수
  ↓
잔여 데이터/결제 확인
  ↓
증빙 보관
```

## 1단계: 신청과 수집

교육생 이메일을 수집한다.

수집 방식은 Google Form, 내부 신청 시스템, 수기 명단 모두 가능하다. 다만 이메일 주소도 개인정보다. 따라서 최소한 다음 내용을 안내한다.

- 무엇을 위해 이메일을 받는지
- 어떤 서비스에 등록할지
- 언제까지 쓸지
- 교육 종료 후 어떻게 처리할지
- 동의하지 않으면 어떤 기능을 못 쓰는지

### 수집 권장 항목

| 항목 | 필수 여부 | 설명 |
|---|---|---|
| 성명 | 필수 | 교육 출석 확인용 |
| 이메일 주소 | 필수 | 서비스 초대용 |
| 소속/과정명 | 선택 또는 필수 | 교육 과정 식별용 |
| 동의 여부 | 필수 | 개인정보 수집·이용 및 국외 이전 고지 확인 |
| 비상 연락처 | 가급적 지양 | 정말 필요할 때만 수집 |

## 2단계: 검증

초대 전에 목록을 정리한다.

확인할 내용은 다음과 같다.

- 이메일 오타
- 중복 이메일
- 개인 Gmail인지 조직 계정인지
- 이미 초대된 계정인지
- 교육 대상자가 맞는지
- 동의 누락자가 있는지

작은 실수도 결제로 이어질 수 있다. 좌석 기반 서비스에서는 잘못 초대한 계정이 곧 비용이 될 수 있다.

## 3단계: 고지와 동의 확인

고지와 동의는 서비스 초대 전에 끝내는 것이 좋다.

특히 ChatGPT, Cursor, Claude 같은 해외 SaaS는 프롬프트, 코드, 첨부파일이 해외 서버로 전송될 수 있다. Privacy Mode나 모델 학습 제외 정책이 있더라도 “전송 자체”는 남는다. 그래서 국외 이전 고지는 별도로 검토해야 한다.

## 4단계: 초대와 라이선스 할당

서비스별로 방식이 다르다.

| 서비스 | 할 일 |
|---|---|
| ChatGPT Business | Workspace settings → Members에서 초대, 역할과 좌석 선택 |
| OpenAI API | 조직 초대 후 프로젝트에 추가, 필요하면 서비스 계정 생성 |
| Cursor | 팀 초대, Privacy Mode 확인, 필요한 경우 SSO 설정 |
| Google Workspace | 사용자 생성, OU 배치, 라이선스 할당 |
| Colab | Colab 서비스 ON 확인, Pro/Pro+ 라이선스 할당 |
| Claude | 조직 초대, Primary Owner와 Owner 설정 확인 |

## 5단계: 접속 테스트

대량 교육 전에 반드시 테스트한다.

테스트 체크리스트는 다음과 같다.

- 초대 메일이 도착하는가.
- 사용자가 수락할 수 있는가.
- 워크스페이스가 올바르게 보이는가.
- 유료 기능이 보이는가.
- API 키 생성 권한이 의도대로 제한되는가.
- Cursor Privacy Mode가 켜져 있는가.
- Colab Pro 기능이 활성화되는가.

## 6단계: 교육 중 운영

교육 중에는 완벽한 통제보다 빠른 대응이 중요하다.

- 로그인 안 됨
- 초대 메일 못 받음
- 다른 워크스페이스로 들어감
- Pro 기능 안 보임
- Cursor가 개인 계정으로 열림
- API 키가 노출됨
- 결제 한도에 걸림
- 사용량이 급증함

이런 문제가 생기면 현장 담당자와 계정 관리자가 바로 연락할 수 있어야 한다.

## 7단계: 권한 회수

교육 종료 후에는 가능한 한 빨리 회수한다.

| 서비스 | 회수 조치 |
|---|---|
| ChatGPT Business | 멤버 제거, 좌석 수 조정, 결제 기준 확인 |
| OpenAI API | 프로젝트 멤버 제거, API 키 삭제, 서비스 계정 제거 |
| Cursor | 팀 멤버 제거, 필요 시 계정 삭제 안내 |
| Google Workspace | 라이선스 회수 또는 계정 정지/삭제 |
| Colab | Colab Pro/Pro+ 라이선스 회수, 서비스 접근 필요 여부 확인 |
| Claude | 조직 멤버 제거, 데이터 export/retention 영향 확인 |

## 8단계: 잔여 데이터와 결제 확인

권한 회수와 데이터 삭제는 같은 말이 아니다.

예를 들어 ChatGPT에서 멤버를 제거해도 사용자의 개인 계정 자체가 삭제되는 것은 아니다. OpenAI 계정 삭제 시에는 데이터 삭제가 30일 내 처리된다는 공식 설명이 있지만, 보안 또는 법적 의무가 있으면 일부 데이터가 더 오래 보관될 수 있다.

Cursor도 계정 삭제 시 30일 내 완전 삭제를 보장한다고 설명하지만, 팀에서 제거하는 것과 사용자 계정 삭제는 다르다.

결제 역시 마찬가지다. 사용자를 제거했다고 이미 결제된 연간 좌석 비용이 바로 돌아오는 것은 아닐 수 있다.

## 9단계: 증빙 보관

다음 자료를 한 폴더에 남긴다.


## 자주 헷갈리는 점

### 초대 메일을 보냈으면 계정 발급이 끝난 것인가?

아니다. 사용자가 초대를 수락해야 실제로 사용할 수 있다.

### 멤버를 제거하면 데이터도 삭제되는가?

항상 그렇지 않다. 서비스마다 다르다. 멤버 제거, 좌석 회수, 계정 삭제, 대화 삭제를 구분해야 한다.

### 교육 종료 후 바로 모든 결제가 멈추는가?

아니다. 월간, 연간, 선결제, true-up, 크레딧 방식에 따라 다르다.

## 운영 팁

회수일을 교육 종료일 다음 날로 잡지 말고, 가능하면 교육 종료 당일에 처리한다. 계정이 하루 더 열려 있으면 그 하루 동안도 사용 가능하다.
