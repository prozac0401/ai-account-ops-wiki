## 개요

AI 계정 운영에서 가장 먼저 해야 할 일은 서비스 이름을 외우는 것이 아니다. **각 서비스가 어떤 층위에 있는지 구분하는 것**이다.

ChatGPT Business와 OpenAI API는 같은 OpenAI 제품군이지만 결제와 권한 체계가 다르다. Google Workspace와 Colab도 같은 Google 관리 콘솔에서 보일 수 있지만, 계정 서비스와 컴퓨팅 라이선스는 다르다.

## 한눈에 보기

| 서비스 | 주 용도 | 과금 방식 | 관리자 포인트 | 가장 흔한 착각 |
|---|---|---|---|---|
| Google Workspace | 조직 계정·메일·관리 콘솔 | 사용자 라이선스 | 사용자, OU(Organizational Unit)<sup class="wiki-footnote-ref"><a class="wiki-footnote-link" id="abbr-ref-1" href="#abbr-note-1">[1]</a></sup>, 서비스 ON/OFF, 결제 | Cloud Identity Free와 Workspace를 같은 상품으로 봄 |
| Colab Pro/Pro+ | 노트북 컴퓨팅 리소스 | 사용자 라이선스 또는 주문 조건 | 서비스 ON + 라이선스 할당 | 라이선스만 사면 바로 쓴다고 생각함 |
| ChatGPT Business | 팀용 ChatGPT 워크스페이스 | 좌석 기반 | 멤버, 역할, 좌석, 결제 | API 사용량도 포함된다고 생각함 |
| OpenAI API Platform | 프로그램에서 모델 호출 | 사용량 기반 | 조직, API 작업 단위, API 키, 예산 | 개인 API 키를 팀 키처럼 씀 |
| Cursor | AI 코드 편집기 | 사용자/팀 구독 + 사용량 요소 | 팀, Privacy Mode, SSO(Single Sign-On)<sup class="wiki-footnote-ref"><a class="wiki-footnote-link" id="abbr-ref-2" href="#abbr-note-2">[2]</a></sup>, 청구 | Privacy Mode면 국외 이전 이슈도 사라진다고 봄 |
| Claude Business/Team | 팀용 LLM(Large Language Model)<sup class="wiki-footnote-ref"><a class="wiki-footnote-link" id="abbr-ref-4" href="#abbr-note-4">[4]</a></sup> 워크스페이스 | 사용자 구독 | Primary Owner, 조직 데이터, 보존 정책 | 멤버 제거가 데이터 삭제라고 생각함 |

## 계정 계층 구조

### Google 계열

```text
Google Admin Console
├─ 조직 계정
├─ Cloud Identity Free
├─ Google Workspace Business Starter 등
└─ Additional Google services
   └─ Colab
      ├─ 무료 Colab 서비스 ON/OFF
      └─ Colab Pro / Pro+ 라이선스
```

핵심은 “사용자 계정”, “서비스 ON/OFF”, “라이선스”가 분리되어 있다는 점이다.

### OpenAI 계열

```text
OpenAI
├─ ChatGPT Business Workspace
│  ├─ Owner / Admin / Member
│  ├─ Standard ChatGPT seat
│  └─ Codex seat
└─ API Platform Organization
   ├─ Organization Owner / Reader
   ├─ API 작업 단위 Owner / Member
   ├─ Service Account
   └─ API Key
```

ChatGPT Business 좌석을 샀다고 API 사용량이 자동으로 무료가 되는 것은 아니다. OpenAI 도움말도 ChatGPT Business와 API Platform은 별도이며 API 사용량은 별도로 청구된다고 설명한다.

### Cursor 계열

```text
Cursor Team / Enterprise
├─ Team Admin
├─ Team Member
├─ Privacy Mode
├─ Usage analytics
├─ SSO(Single Sign-On) / SCIM(System for Cross-domain Identity Management)  [플랜별 차이]
└─ Model provider routing
```

SSO(Single Sign-On)와 SCIM(System for Cross-domain Identity Management)<sup class="wiki-footnote-ref"><a class="wiki-footnote-link" id="abbr-ref-3" href="#abbr-note-3">[3]</a></sup>은 플랜별 제공 범위가 다르다.

Cursor는 코드를 다룬다. 그래서 일반 챗봇보다 입력 금지 데이터 안내가 더 중요하다. 하드코딩된 비밀키, 고객정보가 들어간 코드, 비공개 레포 전체를 그대로 넣는 일이 생기기 쉽다.

## 교육 운영 기준으로 묶기

### 교육 전에 필요한 것

- 서비스별 구매 가능 여부 확인
- 결제 수단 확인
- 수강생 이메일 수집
- 개인정보 고지·동의
- 보안 검토
- 초대 테스트

### 교육 중 필요한 것

- 접속 문제 대응
- 좌석 부족 대응
- 결제 실패 대응
- Pro 기능 미노출 대응
- 금지 데이터 입력 안내
- 사고 발생 시 즉시 차단

### 교육 후 필요한 것

- 멤버 제거
- 라이선스 회수
- API 키 폐기
- 잔여 결제 확인
- 데이터 삭제 가능 범위 확인
- 회수 결과 로그 보관

## 서비스별 “최소 운영 단위”

| 서비스 | 최소 운영 단위 |
|---|---|
| Google Workspace | 사용자 계정 1개 + 라이선스 1개 |
| Colab | 사용자 1명 + Colab 서비스 접근 + Pro/Pro+ 라이선스 |
| ChatGPT Business | 워크스페이스 1개 + 표준 좌석 최소 수량 + 멤버 |
| OpenAI API | 조직 1개 + 교육용 API 작업 단위 1개 + API 키 또는 서비스 계정 |
| Cursor | 팀 1개 + 팀 멤버 + Privacy Mode 정책 |
| Claude | 조직 1개 + 사용자 + Primary Owner 관리 정책 |

## 운영자가 기억할 만한 문장

- ChatGPT는 사람에게 주는 도구다.
- OpenAI API는 시스템에 주는 도구다.
- Cursor는 코드에 가까운 도구다.
- Colab은 컴퓨팅 자원에 가까운 도구다.
- Google Workspace는 계정과 결제의 바닥이다.

이 구분만 잡아도 운영 실수가 많이 줄어든다.
