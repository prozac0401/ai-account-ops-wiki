## 개요

이 문서는 교육용 생성형 AI 계정을 **받고, 열어 주고, 쓰게 하고, 회수하고, 증빙을 남기는 일**을 정리한다.

처음에는 Google Workspace와 Colab 중심으로 시작했다. 이후 ChatGPT Business, Cursor, OpenAI API, Claude Business 같은 도구까지 들어오면서 단순한 “계정 발급”이 아니라 **AI 계정 운영 체계**가 되었다.

## 왜 별도 위키가 필요한가

생성형 AI 서비스는 기능보다 운영이 더 헷갈린다.

예를 들어 다음 질문이 한꺼번에 나온다.

- 수강생 Gmail 주소를 Google Form으로 받아도 되는가.
- ChatGPT 팀 계정에 초대하면 개인정보 국외 이전 동의를 받아야 하는가.
- Cursor는 Privacy Mode가 있으면 동의가 필요 없는가.
- Cloud Identity Free 조직에서 Business Starter를 붙이면 기존 계정은 어떻게 되는가.
- Colab Pro 라이선스는 샀는데 왜 사용자가 Pro 기능을 못 쓰는가.
- OpenAI API 키는 개인 키로 나눠 써도 되는가.
- 교육이 끝나면 멤버만 제거하면 데이터가 삭제되는가.
- 카드 결제, 선결제, 인보이스는 누가 관리해야 하는가.

이 위키는 이런 질문을 “서비스별 설명”이 아니라 **운영 흐름** 기준으로 묶는다.

## 운영의 실제 전제

기존 질의응답을 기준으로 보면 운영 전제는 대략 다음과 같다.

| 항목 | 내용 |
|---|---|
| 수강생 계정 | 개인 Gmail 또는 조직 Google 계정이 혼재 가능 |
| 계정 수집 | Google Form으로 이메일 주소 수집 가능 |
| 주요 서비스 | ChatGPT Business, Cursor, Google Workspace, Colab, OpenAI API |
| 사용 환경 | 교육장 실습, 경우에 따라 보안 규제가 비교적 완화된 환경 |
| 핵심 통제 | 안내, 동의, 금지 데이터 고지, 초대/회수 로그, 결제 증빙 |
| 법무·보안 검토 | 개인정보 수집·이용, 국외 이전, 제3자 제공, 보안 사고 대응 |

## 문서가 다루는 범위

### 포함한다

- 계정 등록과 회수 절차
- 관리자와 교육 담당자의 역할 분담
- 수강생 안내 사항
- 서비스별 과금 구조
- 카드 결제와 선결제 이슈
- 개인정보 동의서와 고지서 초안
- 국외 이전과 데이터 리텐션
- 실제 질의응답 기반 FAQ(Frequently Asked Questions)

### 포함하지 않는다

- 각 AI 모델의 성능 비교
- 프롬프트 작성법 전체
- 교육 커리큘럼 설계 전체
- 법무 최종 의견
- 회계 처리 기준의 최종 판단

## 운영상 가장 중요한 문장

> “사용하게 해 주는 것”과 “안전하게 쓰게 하는 것”은 다르다.

계정 운영자는 둘 다 챙겨야 한다. 특히 교육 현장에서는 초대보다 회수가 더 중요하다. 초대는 링크 한 번이면 되지만, 회수와 삭제는 서비스마다 다르게 움직인다.
